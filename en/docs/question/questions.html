# Deployment scenario problems

# Deploy to nginx to access 400

Cause 1: The backend configuration permissions are too many, exceeding the default maximum value of nginx configuration Solution: Modify the nginx configuration

http { client_ header_ buffer_ size 64k; # Default value 1k large_ client_ header_ buffers 4 64k; # Default value 4 8k }

# Failed to deploy to iis PUT, POST and other requests 405 exception

An HTTP Error 405.0 - Method Not Allowed error occurred when requesting the Put/POST/DELETE operation of the REST service deployed on IIS7.5.

Removing IIS's rejection of PUT and DELETE requests from HttpModule is caused by a custom HttpModule named WebDAVModule registered by IIS for the website by default. If our site does not need to provide support for WebDAV, the most direct way for IIS7 to close WebDAV extension service is to remove the registered HttpModule by using the following configuration:

<system.webServer>
<modules>
<remove name="WebDAVModule" />
</modules>
<handlers>
<remove name="WebDAV" />
</handlers>
</system.webServer>

Remove the WebDAV module Find the WebDAVModule from the module and remove it

# Error reported by the startup project sqlsugar

Question: An exception of type "SqlSugar. SqlSugar Exception" occurred in SqlSugar. dll, but it was not processed in the user code. Chinese prompt: An error occurred in the process of connecting to the database. Check whether the server is normal and whether the connection string is correct. If you can't find the reason, please first Google the error message

Please check whether the string connection is correct

# Disable script

Error content: npm: Unable to load file D: nodejs node_ Global npm.ps1 because scripts are not allowed to run on this system.

Because scripts are not allowed to run on this system, that is, there is no permission. Let's see:

d:> get-ExecutionPolicy

Restricted Restricted means that the status is prohibited, so give a permission: Set-ExecutionPolicy - Scope CurrentUser Input: RemoteSigned

# Error executing SQL

For example, the column name 'CoverUrl' is invalid. Generally, this error will be prompted when the database field is updated. Add the corresponding field to the database table.
